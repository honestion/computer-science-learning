# **DRY rule**
**DRY** stands for **"Don't Repeat Yourself"** that literally means that I shouldn't repeat yourself, so every logic block
should appear you in your code only once
Consequently it means that you should separate logic blocks with conditionals and general actions like "input(), print()" and others
That method is literally used in functions and variables
### **Refactoring** - is the process where you improve your codes sturcture without changing its functions
